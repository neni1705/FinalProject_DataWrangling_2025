# Final Project: Data Wrangling 2025

 EAS diberikan dalam bentuk final project, yang dikerjakan secara berkelompok (1 tim terdiri 3-4 orang)
Proyek akhir ini memberikan pengalaman langsung dalam melakukan cleaning, standardizing, wrangling, dan analisis data dunia nyata menggunakan Python.
Anda akan:
1.	Memilih satu dataset
2.	Membangun pipeline pembersihan data dari awal hingga akhir
3.	Menyusun Notebook analisis utama termasuk Exploratory Data Analysis (EDA)
4.	Mengumpulkan seluruhnya melalui GitHub, kemudian link Github di upload (oleh perwakilan kelompok) di MyClassroom 
Proyek ini dirancang menyerupai alur kerja data scientist profesional di industri.

Repositori GitHub Anda harus mengikuti struktur berikut:
```
final-project/
│
├── data/
│   ├── raw/                 # Dataset mentah
│   └── processed/           # Dataset setelah dibersihkan
│
├── notebooks/
│   └── final_project.ipynb  # Notebook analisis utama
│
├── requirements.txt         # Daftar package Python yang digunakan
└── README.md                # File yang menjelaskan final project
```
Pilih satu dari dataset berikut (akses data dari link Github berikut):
i.	Hotel Bookings Data
o	Data pemesanan hotel (kapan, lama menginap, pembatalan, dsb.)
ii.	NFL Attendance Data
o	Data kehadiran penonton pertandingan American football (NFL)
iii.	Spotify Genre Data
o	Fitur-fitur lagu (danceability, energy, valence, dll.) per genre musik
Anda akan menulis laporan dalam bentuk Jupyter Notebook (.ipynb) yang memuat bagian-bagian sesuai rubrik penilaian di bawah.
A.	Pendahuluan
•	Deskripsi dataset
•	Alasan dataset menarik
•	Pertanyaan/tujuan analisis
B.	Setup & Package
•	Semua import di bagian atas notebook
•	Path file disimpan dalam variabel
•	Penjelasan singkat fungsi paket
C.	Data Preparation
•	Import dataset
•	Langkah cleaning dan data wrangling dengan penjelasan Markdown
•	Standardisasi & transformasi
•	Save dataset bersih ke data/processed/
D.	Exploratory Data Analysis
Minimal:
•	3 visualisasi
•	2 tabel ringkasan
•	Interpretasi untuk setiap output
E.	Ringkasan & Kesimpulan
Diskusikan:
•	Temuan utama
•	Keterbatasan
•	Saran pengembangan

Persyaratan Reproducibility (WAJIB)
Proyek harus bisa dijalankan ulang tanpa error.
a.	Notebook harus:
•	Berjalan dari atas ke bawah tanpa error
•	Menggunakan path relatif (mengakses file berdasarkan posisi file tersebut di dalam folder proyek, bukan menggunakan path lengkap yang mengarah ke lokasi di komputer Anda)
•	Menyimpan hasil cleaning otomatis
b.	Repo harus berisi: requirements.txt
Contoh:
pandas
numpy
matplotlib
seaborn
c.	Dokumentasi cleanup harus menjelaskan:
•	Cara menjalankan ulang proyek
•	Setup environment
•	Struktur file


Sebelum submit, pastikan pada GitHub
•	Struktur folder benar
•	Notebook berjalan tanpa error
•	Dataset bersih tersimpan
•	requirements.txt lengkap


Jika ada pertanyaan, tanyakan lebih awal.
